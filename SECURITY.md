# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in SponsorBlock AI, please report it privately
rather than opening a public issue. This allows us to address the issue before it is
publicly known.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

To report a vulnerability, please contact the maintainers directly. We aim to respond
within 48 hours and provide a timeline for a fix within 7 days.

## Scope

Security issues that are in scope include:

- Unintended exposure of user API keys
- Vulnerabilities that allow remote code execution
- XSS or injection vulnerabilities in the popup or content scripts
- Privilege escalation or permission bypass
- Man-in-the-middle vulnerabilities in API communication

## Out of Scope

- Issues in the user's own LLM provider or API endpoint
- Issues in the SponsorBlock API itself
- Social engineering attacks requiring user interaction
